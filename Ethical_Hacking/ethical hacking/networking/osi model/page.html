Understanding The OSI Model
  • This is where all the different models come together
  • You should really want to undersdtand the model and how it works
  • Some people may just reference by the layers - say layer 2 instead of a switch, or layer 3 instead of router

We can use a mnemonic system to remember the diferent layers of an OSI Model.
  • Please
  • Do
  • Not
  • Throw
  • Sausage
  • Pizza
  • Away

This can be translated as the steps within the OSI Model:
  • 1) Physical - data cables, cat6 cables, something you would plug in
  • 2) Data - switching, MAC addresses
  • 3) Network - IP addresses, routing
  • 4) Transport - TCP/UDP
  • 5) Session - session management
  • 6) Presentation - WMV, JPEG, MOV (media)
  • 7) Application - HTTP, SMTP, applications that you use

With the OSI Model when we recieve data, the data passes through the physical layer to the application (consecutively).
When we send data, it passes through the application to the physical layer.

You will always want to start from the beginning and work your way to the end when debugging or tryign to find a problem
  • You go through every layer starting from the first (physical) down to application level

So if you're on a help desk and someone says they have a problem with their router (a layer 2), you would get them to check the cables first,
then check the switches & MAC address and work your way down.  You wouldn't start from layer 7 and work your way down to layer 1. 

REMEMBER !!

  • Learn the language
  • Remember the mnemonic version of the layers (please do not throw sausage pizza away)
