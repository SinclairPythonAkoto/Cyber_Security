Passive Recon Overview

This is a diagram to show the different examples of passive recon.  Passive recon is when we gather data from public information that is already out there.

You may be tasked with a physical assessment so these are the type of things you would look for.

This is a quick overview of how we would recon a web / host address which we will cover in the course.


Target Validation
  • First thing is to validate your target address - this is the correct address for the target?
  • If your are given an incorrect address you can validate the address before you start any exploits.

Finding Subdomains
  • Use tools like Google Fu & Nmap to browse for subdomians - more will be covered in the course

Fingerprinting 
  • What is running on a website / host?  What type of services are out there
  • If they are running a web server what version is it?
  • What ports do they have open on their machine?
  • Do they have any ports open?

Data Breaches
  • The most common external way to get into networks
  • Cover breached incidents from the past that have leaked data
  • Use data dumps to get access

The better your information gathering and scanning/enumeration skills are, the better pen-tester you will be!  It is really important to master the first 2 stages of the 5 stages of ethical hacking.
