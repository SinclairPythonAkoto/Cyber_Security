Ethernet
Ethernet is a collection of network protocols & standards.  Because of this, Ethernet is not defined by one singluar protocol or standard.  For the CCNA course, we will look at the types of cables defined by Ethernet standards.


Using network standards protocols help other computers communicate because by using the same protocol/standard then the computers can communicate and understand eachother even if they are using different languages / configuration / devices, etc.

The two can communicate and understand each other using Ethernet protocols/standards

The hardware for Ethernet cables arre set to a universal standard, so you would not be able to apply it an uncompatable device.

The RJ-45 will not be able to fit in the USB slot

Manufacturers apply the universal standards to the hardware (ports, cables, connectors, etc), just how standards are applied to network IP addresses - different types of end hosts stick to the same networking rules.


Bits and Bytes
Connections between devices in a network operate at a set speed.  The speed is measured in bits per second (Bps).
So what is a bit?  It is a binary digit represented by a 0 or 1.  Computers, websites, operating systems, etc all work off from binary code.  8x bits make up 1x byte.


As mentioned before, bits are measured in bits per second.  This is because when traffic/data is being sent across a network, it is sent across one bit at a time, not one byte at a time.

UTP Cables
UTP (Unshielded Twisted Pair) cables are cables used to comply with Ethernet standards.  They also protect against EMI (Electromagnetic Interference).

We should note not all Ethernet standards use 8x wires
  • Unshielded means they have no metalic sheild.  This can make them vulnerable to electrical interference.
  • The twisted wires helps prevent electrical interference.
  • The wires are twsited in pairs; in this picture there are 4x pairs, meaning 8x cables.

If we study the picture below we can see the different UTP cables use different amount of wires.

We will look deeper into 10BASE-T & 100BASE-T cables.


Full-Duplex
We will look at the connectivity between an end host and a switch.  Imagine a scenario where we are connecting a PC to a switch with 10BASE-T or 100BASE-T cables.  The numbers in the diagram represent the pins on the RJ-45 connector.

The lines are straight but the wires would be twisted togteher in the physical hardware

The PCs (end host) will transmit data through pins 1 & 2, and the switch interface will recieve the data through pins 1 & 2.
On the switch, pins 3 & 6 are used to transmit the data, while the end host recieves the data through pins 3 & 6.

This is a Full-Duplex transmission because both devices can send & recieve data at the same time without any data colliosions.  This is achieved by using seperate wires.


Straight-through
We will now look at the connectivity between a router and a switch, using 10BASE-T or 100BASE-T cables.

This also uses a Full-Duplex transmission

A router transmits data on pins 1 & 2, and recieves data through pins 3 & 6.
The switch recieves data on pins 1 & 2, and transmits data through 3 & 6.

These are also called straight-through cables because the pins are connected to their matching numbers (pin 1 connects to pin 1, pin 2 connects to pin 2, etc).


Crossover cable
Sometimes we may want to connect two of the same type of  network nodes together (ie, two switches, two PCs, etc).
We will look at how we can connect two switches that are the same, using 10BASE-T or 100BASE-T cables.

If we try to use the Full-Duplex method, we can see that we cannot transmit data because pins 3 & 6 are not set up to recieve data on a switch (and also the reciever pins are incorrectly set up).

The cables cannot send / recieve data this way

To fix this we connect the pins to their opposite counterparts on the other side.  

You can also connect a router to a end host using crossover cables and it would still work as normal.

Pins 1 & 2 on the left will connect to pins 3 & 6 on the right, pins  3 & 6 will connect to 1 & 2

The wires are crossed over each other - hence the name Crossover cable.  The transmit pins on one side is connected to the recive pins on the oethr side.

Auto MDI-X
This automatically detects which pin the transmitting data is coming from, and then adjust to their pins to recieve  & transmit the data.

More modern cables have this but is still good practice to know the straight-throu gh cables and crossover cables.

Take that if an Auto MDI-X is connected to an old network node then the auto detect function will not work.


1000BASE-T or 10GBASE-T cables
Now we will cover the 1000BASE-T or 10GBASE-T cables; they have two main differences - they use all 8x wires and they are bidrectional (each pair can send & recieve data).

This is the reason why they operate at much higher speeds.

Fiber-optic cables

Instead of sending an electrical signal through copper wiring, they send light over glass fibers.  


On each end of the cable there are 2x connectors to enable the capability to recieve & send data.

Let's take a look at the structure of a fiber optics cable.

  • The fiberglass core: light is transmitted down the core to transmit one device to another
  • Cladding reflects light
  • Protective buffer : protects the fiberglass from breaking
  • Outer jacket protects against the elements

There are two main types of fiberglass cables - single-mode fiber & multi-mode fiber cables.

REMEMBER !!
  • Ports are also known as interfaces
  • Speed is measured in bits per second (Bps, Kbs, Mbs, Gbs, etc)
  • Data is sent one bit at a time, in binary digits - 8x bits amounting to 1x byte to create a 0 or 1.
  • UTP cables are used in Ethernet standards.  They are the standard for wired connections to switches.
  • Full-Duplex transmission allows network nodes to send and recive data at the same time
  • Connectivity between an end host to a switch and a router to a switch are the same.  Pins 1 & 2 transmit while pins 3 & 6 recieve.
  • Stright-through cable are pins that connect to the same number.
  • Crossover cables do not connect the pins to the same number but rather their opposites.  The transmission on one side will connect to the reciever on the other side
  • Auto MDI-X will detect which pin the data is coming from then adjusts its pin accordingly to recieve and send the data.
  • Fiber-Optic cables have a longer distance than UTP cables and are protected against EMI.
  • UTP does not support for distances over 100m.
  • The SFP ports for the fiber-optic cables are more expensive than RJ45 ports,
